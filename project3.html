Template Changes

I used Bootstrap templates to build this portfolio website in order to learn more about manipulating HTML and CSS code to achieve the desired format and content. I started off with the Clean Blog template.

Additions & Subtractions

I added new pages, an accordion, and different styles (change in backgrounds, color scheme, page transitions, footer, social media icons at the bottom, and buttons) to get the end result that you see now. I removed the contact form and provided my email address and LinkedIn URL instead so that the form of communication would be more personal for the user and the user would be be able to keep track of when they sent their message as well.

Website Description

My portfolio website has three main web pages (home, about, and contact) and six project pages.

For navigation, there is one menu item for each of the main web pages and a responsive pagination at the bottom of each product page.

For visual design, I stuck with a simple color scheme and consistent page layouts. I also sparingly utilized relevant and eye-catching graphics and various design elements to make the site more appealing. I aimed to make sure the content is informative, easy to read, and as concise as possible.

For search engine optimization, although I don't expect my portfolio website to be searched on search engines very much, I still included a concise and actionable meta description that is unique to each page.

What I Learned

While building my website, I also learned:

  • How to inspect code to troubleshoot issues.
  • How to use GitHub to host my website and use Github Desktop to commit or discard changes, keep a log of changes, help me organize my files, and push local commits to origin remote.

I now feel more comfortable using HTML tags and attributes and CSS styles.

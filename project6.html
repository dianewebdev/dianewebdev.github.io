Issues

One of the issues I encountered while setting up the store was that when it attempted to generate outgoing emails to customers and administrators, it produced mail failure errors from WordPress. To troubleshoot this, I used a plugin called WP Mail SMTP. By default, WordPress uses the PHP mail function to send its emails and if the hosting provider does not update the PHP on the servers regularly, problems with sending out automated or manual emails to users from WordPress can occur. To fix this, I created a web application associated with the organization's Gmail using the Google Cloud platform. I then used it to configure Gmail as the email provider that handles all of the outgoing emails from WooCommerce. This way, their website would be using Simple Mail Transfer Protocol rather than the PHP mail function. This solution was successful and my client appreciated me taking the extra steps to fix the issue, especially since sending out emails were a significant part of their marketing strategy.
